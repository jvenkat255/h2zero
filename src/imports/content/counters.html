Counters are simple SQL statements that return one, and only one integer value.

This is a light-weight way to get a count of something without going through a selectClause bean finder.

The key counters is a JSON Array on the table JSON object, for example:

				
"tables": [{\n}
{\t}{{{\n}
{\t}{\t}"fields": [{\n}
{\t}{\t}{\t}...{\n}
{\t}{\t}],{\n}
{\t}{\t}"counters": [{\n}
{\t}{\t}{\t}...{\n}
{\t}{\t}]{\n}
{\t}}{\n}
]{\n}
{import src/imports/tables/allowable-keys-table.templar}

Automatically generated methods

Every table automatically has countAll methods generated, with the following signatures: 

				
public static int countAll() throws SQLException{\n}
public static int countAll(Connection connection) throws SQLException{\n}
public static int countAllSilent(){\n}
public static int countAllSilent(Connection connection){\n}

Example 1: Simple Count

				
"counters": [{\n}
{\t}{{{\n}
{\t}{\t}"name": "countNumberOfUsers",{\n}
{\t}{\t}"selectClause": "select count(*) from user"{\n}
{\t}}{\n}
]{\n}

As it would appear, this would return the number of users in the table. Without a {import src/imports/links/whereClause.templar}, these methods are identical to the countAll methods and consequently aren't necessary, however this is included for illustrative purposes.

Example 4: Parameterised Counter with in: designator fields

				
"questions": [{\n}
{\t}{{{\n}
{\t}{\t}"name": "countUsersInAges",{\n}
{\t}{\t}"selectClause": "select count(*) > 0 from user",{\n}
{\t}{\t}"whereClause": "where num_age in (...)",{\n}
{\t}{\t}"whereFields": [{\n}
{\t}{\t}{\t}"in:num_age"{\n}
{\t}{\t}]{\n}
{\t}}{\n}
]{\n}

In the above example, the count being given is the number of users that have an age that equals that of the passed in list

Note both the {import src/imports/links/whereClause.templar} having the ellipses (...) to designate that this is an in query and the {import src/imports/links/whereFields.templar} using the specialised in: designator.

The code generation uses these two parts in order to correctly specify the method parameters and the way in which a PreparedStatement is constructed.
