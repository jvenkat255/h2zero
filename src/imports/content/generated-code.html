/src/main/java/</your/package/here>

This is where the majority of the code will be generated to, </your/package/here> is the java packaging and directory structure as defined in the .h2zero file. In the example file, the package is defined as: 

	
		"package": "synapticloop.sample.h2zero"

Which is a key under the {import src/imports/links/database.templar} JSON object.

It is recommended that the package is namespaced with a .h2zero as the last hierarchy to separate out the generated code.

This will generate code that goes into the the following directory 

	
/src/main/java/synapticloop/sample/h2zero

The /bean directory

The bean directory contains all of the code that is generated for the {import src/imports/links/selectClause.templar} key. The beans are a sub-set of the fields from one or more tables.

Code is only generated if there is a {import src/imports/links/selectClause.templar} generated for any of the beans.

The /counter directory

The counter directory contains all of the code that is generated for the {import src/imports/links/counters.templar} key.

The /deleter directory

The deleter directory contains all of the code that is generated for the {import src/imports/links/deleters.templar} key.

The /finder directory

The finder directory contains all of the code that is generated for the {import src/imports/links/finders.templar} key.

The /inserter directory

The inserter directory contains all of the code that is generated for the {import src/imports/links/inserters.templar} key.

The /model directory

The model directory contains all of the code that is generated for the {import src/imports/links/model.templar} key. The model is a POJO

The /question directory

The question directory contains all of the code that is generated for the {import src/imports/links/questions.templar} key.

The /taglib directory

The taglib directory contains all of the code that is generated for the {import src/imports/links/taglib.templar} key.

Within this directory are sub-directories of all of the table objects, with their individual tags.

The /updater directory

The updater directory contains all of the code that is generated for the {import src/imports/links/updaters.templar} key.

Option code generation

Option Generated code notes
formBeans Genartion of beans for form completion Under development
adminpages Administrative pages for database CRUD opertaions Under development
taglib Tag libraries for finders, counters and questions Automatically generated for each of the SQL interactors, this will also automatically generate the correct tag library desciptor (tld) file.
jsp unused Under development
java Generate the model beans Automatically generate the POJOs for the tables, including finders, deleters, inserters, etc.
sql Generate the database cration script Automatically generate the SQL database creation script and any constants table inserters as well.
