Field updaters are a simple way to generate singular updaters that update a single field for either the complete table, or a specific row identified by its primary key.

The key fieldUpdaters is a JSON Array of JSON Objects on the table JSON object, for example:

	
"tables": [{\n}
{\t}{{{\n}
{\t}{\t}"fields": [{\n}
{\t}{\t}{\t}...{\n}
{\t}{\t}],{\n}
{\t}{\t}"fieldUpdaters": [{\n}
{\t}{\t}{\t}...{\n}
{\t}{\t}]{\n}
{\t}}{\n}
]{\n}

Adding the following fieldUpdaters JSON array of JSON Objects:

				
"fieldUpdaters": [{\n}
{\t}{{ "fl_is_indexed": "unique" },{\n}
{\t}{{ "fl_is_alive": "multiple" }{\n}
]{\n}

Assuming that the primary key is named id_user

unique updaters: The name of the updater will be update<CamelCaseFieldName>By<CamelCase<PrimaryKeyFieldName>>

multiple updaters: The name of the updater will be update<CamelCaseFieldName>

This will automatically generate two updaters:

  1. The first updater is updateFlIsIndexedByIdUser which will return the number of rows updated (which should always by one) with two parameters passed into the method, namely flIsIndexed, and idUser.
  2. The second updater is updateFlIsAlive. which will the number of rows updated and has one parameter passed into the method, namely flIsAlive.

    3. The alternate way to define the above updaters is as follows (which would generate identical code): 

    				
"updaters": [{\n}
{\t}{{{\n}
{\t}{\t}"name": "updateFlIsIndexedByIdUser",{\n}
{\t}{\t}"setClause": "set fl_is_indexed = ?"",{\n}
{\t}{\t}"setFields": [{\n}
{\t}{\t}{\t}"fl_is_indexed"{\n}
{\t}{\t}],{\n}
{\t}{\t}"whereClause": "where id_user = ?",{\n}
{\t}{\t}"whereFields": [{\n}
{\t}{\t}{\t}"id_user"{\n}
{\t}{\t}]{\n}
{\t}},{\n}
{\t}{{{\n}
{\t}{\t}"name": "updateFlIsAlive",{\n}
{\t}{\t}"setClause": "set fl_is_alive = ?"",{\n}
{\t}{\t}"setFields": [{\n}
{\t}{\t}{\t}"fl_is_alive"{\n}
{\t}{\t}]{\n}
{\t}}{\n}
]{\n}

    Which as you can see is a lot more work, more prone to error and un-necessary for something that can be automatically generated.
